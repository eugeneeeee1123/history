# 史阅 THE ARCHIVUM — 待补充内容清单

> 按优先级从高到低排列。P0 影响可用性，建议最先处理；P3 是锦上添花，有余力再做。

---

## P0 · 可用性缺口（建议最先做）

- [ ] **联系表单占位邮箱**：`contact.js` 里的 `archive@thearchivum.dev` 是占位符，上线前务必换成你的真实邮箱，否则 `mailto:` 发不出去。
- [ ] **移动端导航折叠**：现在顶栏是 首页/亚洲/欧洲/美洲/画廊/关于/联系 + 搜索 + 语言切换，共 9 个控件，窄屏挤在一起会换行很挤。建议 <768px 时收进一个汉堡菜单。
- [ ] **人物头像占位**：现在所有人物用同一套"印章 SVG + 姓氏首字"代替头像，121 人视觉上区分度不够。至少可以给评级 ★★★★★ 的头部人物（成吉思汗、恺撒、拉美西斯二世等）配一张线描/剪影插画。
- [ ] **抽屉的键盘可达性**：人物档案 / 时代详情从右侧滑出时，Tab 键还是能跳到背景内容里，且没有做 focus trap；键盘用户和屏幕阅读器体验不完整。
- [ ] **自定义 404 页面**：目前访问不存在的路径会掉到浏览器默认空白页，建议加一个 `404.html`，风格统一、带"返回首页"链接。

## P1 · 内容广度与深度

- [ ] **地域覆盖不全**：目前完全没有非洲（撒哈拉以南）、大洋洲、南亚/印度次大陆，美洲这一卷也漏了**印加帝国**（安第斯文明），只有中美洲的玛雅/阿兹特克。
- [ ] **人物分布严重不均**：东亚 78 人 vs 阿拉伯与埃及 9 人 / 马来与群岛 12 人 / 欧洲 13 人 / 美洲 9 人。亚洲卷内部也是"中国独大"，阿拉伯和马来两条线明显单薄，建议优先补充这两条线的时代与人物。
- [ ] **史料来源标注缺失**：现在每位人物的"生平"是历史重述、"名言"很多是后世归因或合理重写，但没有标注参考资料出处。建议给 `data.js` 里每条人物加一个可选的 `sources` 字段，哪怕只是"参考：《史记》""参考：维基百科"这种粗粒度标注，增加可信度。
- [ ] **时代缺地图**：现在每个时代只有文字描述，没有地理位置概念。可以给每个时代配一张简单的区域示意图（哪怕是用现有的 SVG 印章风格画一张极简地图，标出该时代的核心地理范围）。

## P2 · 功能增强

- [ ] **收藏 / 书签**：用 `localStorage` 让用户在人物抽屉里点一下"收藏"，画廊页加一个"我的收藏"筛选项，方便回顾看过的人物。
- [ ] **人物关系图**：像"刘备—关羽—张飞—曹操"这种君臣/对抗/结拜关系，用简单的力导向图或手绘连线可视化会比纯文字更有记忆点，尤其是三国、战国这类人物关系密集的时代。
- [ ] **人物对比工具**：选两位人物并排比较朝代、评级、身份、代表事迹——类似"关羽 vs 吕布"这种趣味对比，适合作为分享点。
- [ ] **深浅色主题切换**：你最早给的参考图里其实有一版浅色（米白纸感）设计，现在网站只有深色一种。加一个主题切换按钮，两套配色都能保留原来的"档案感"。
- [ ] **正文双语化**：现在英文切换只翻译了导航、按钮这些界面文字，人物的生平/名言仍然是中文。如果真的要给英文读者看，需要把 121 条 `bio` / `quote` 实际翻译（工作量大，建议先挑评级最高的 20-30 位做双语试点）。
- [ ] **拼音/模糊搜索**：现在搜索是精确子字符串匹配，输入"成吉思汗"能搜到，但输入"cheng ji si han"或打错一个字就搜不到。加拼音首字母匹配对移动端用户更友好。

## P3 · 打磨与长期运营

- [ ] **SEO 与结构化数据**：给人物和时代内容加 `schema.org/Person` 标记，写 `sitemap.xml` / `robots.txt`，目前作为纯前端 SPA 式的多页站点，搜索引擎收录效果一般。
- [ ] **`data.js` 拆分懒加载**：现在单文件约 260KB，随便一个页面都要整份加载。未来人物继续增加，可以按大洲拆成 `data-asia.js` / `data-europe.js` / `data-americas.js`，各区页面只加载自己需要的部分。
- [ ] **动效打磨**：时代卡片进入视口时的渐显动画、页面切换的过渡效果，现在都还是"瞬间出现"，可以用 `IntersectionObserver` 加一点节制的滚动动画。
- [ ] **隐私友好的访问统计**：接入 Plausible / Umami 这类不追踪个人信息的统计工具，了解哪些人物、哪个大洲最受欢迎，指导后续内容优先级。
- [ ] **轻量后端**：如果以后想要联系表单真正落库、或加评论/纠错提交功能，需要一个小后端（比如 Cloudflare Worker 转发邮件），目前纯前端只能靠 `mailto:`。
- [ ] **"历史上的今天"小组件**：首页加一个小模块，根据当天日期显示档案里对应的历史事件（需要先给每条大事记年份补上月/日，目前只有年份颗粒度）。
- [ ] **人物档案导出**：在人物抽屉里加一个"导出为图片/PDF"按钮，方便分享单个人物的卡片。

---

### 关于分类准确性的小提醒

人物的"统治者/军事/谋士/传说/其他"分类是关键词自动打的标（`data.js` 生成脚本按称号里的字词判断），不是逐个人工核对的，个别人物归类可能不准，有空的话建议抽查一遍 `data.js` 里的 `cat` 字段。
